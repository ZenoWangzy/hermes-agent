# cron/ - Scheduled Job System

> Auto-generated by init-architect at 2026-04-12T01:49:51

## Module Purpose

Built-in cron scheduler for running automated tasks at specified intervals. Jobs are defined in `~/.hermes/config.yaml` under the `cron` key and can deliver results to any configured messaging platform.

## Entry Point

- Triggered by gateway via `scheduler.tick()` every 60 seconds from a background thread
- CLI: `hermes cron list`, `hermes cron status`

## Key Files

| File | Description |
|------|-------------|
| `jobs.py` | Job definitions: `get_due_jobs()`, `mark_job_run()`, `save_job_output()`, `advance_next_run()`. |
| `scheduler.py` | `tick()` function: checks for due jobs, executes them, file-based lock for single-tick guarantee. |

## Delivery Platforms

Supports 15+ platforms: telegram, discord, slack, whatsapp, signal, matrix, mattermost, homeassistant, dingtalk, feishu, wecom, weixin, sms, email, webhook, bluebubbles.

## Dependencies

- `croniter>=6.0.0` (optional, via `[cron]` extra)
- `hermes_cli.config.load_config`

## Concurrency Notes

- `scheduler.tick()` uses a **file-based lock** to guarantee only one tick runs at a time (prevents overlapping cron runs)
- Tick is called every 60 seconds from a gateway background thread
- Jobs execute synchronously within the tick — long-running jobs block subsequent ticks
