## Module Purpose

Contains all tool implementations for the Hermes agent. Each tool file self-registers its schema, handler, and metadata via `registry.register()` at module import time. The central `registry.py` singleton collects everything; `model_tools.py` triggers discovery and provides the public API.

## Architecture

```
tools/registry.py          (singleton, no deps)
    ^   tools/*.py          (each calls registry.register() at import time)
    ^   model_tools.py      (imports tools.registry + all tool modules)
    ^   run_agent.py, cli.py, batch_runner.py, environments/
```

## Tests

`tests/tools/` contains 121 test files covering every tool, environment backend, browser provider, and edge case.

```bash
python -m pytest tests/tools/ -q                    # All tool tests
python -m pytest tests/tools/test_terminal_tool.py -q  # Specific tool
```

### Adding a Tool Test

1. Create `tests/tools/test_<tool_name>.py`
2. Use `_isolate_hermes_home` fixture (autouse) to prevent writing to `~/.hermes/`
3. Mock external APIs; test handler returns valid JSON string
4. Test availability check (`check_fn`) with and without required env vars
