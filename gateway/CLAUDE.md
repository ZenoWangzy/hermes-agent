## Module Purpose

Multi-platform messaging gateway that allows users to interact with the Hermes agent through Telegram, Discord, Slack, WhatsApp, Signal, Email, and 15+ other platforms from a single gateway process. Handles session management, message dispatch, delivery, hooks, and platform-specific adapters.

## Entry and Startup

- **Entry**: `gateway/run.py` -- `GatewayRunner` class, starts all configured platform adapters
- **CLI trigger**: `hermes gateway` or `hermes gateway start`
- **Config**: Platform settings in `~/.hermes/config.yaml` under `gateway` key

## Architecture

The gateway runs as an async event loop. Each platform adapter:
1. Connects during startup (with token lock for profile safety)
2. Receives messages -> dispatches to `AIAgent` -> returns response
3. Handles slash commands (shared with CLI via `COMMAND_REGISTRY`)
4. Manages per-chat sessions via `SessionStore`

## Adding a New Platform Adapter

1. Create `gateway/platforms/<platform>.py` inheriting from `base.py`
2. Implement required methods: `connect()`, `disconnect()`, `send_message()`, `receive_message()`
3. Register in `gateway/platforms/__init__.py` platform registry
4. Add config schema in `hermes_cli/config.py` under `gateway` defaults
5. Add credential env var to `hermes_cli/auth.py`
6. Write tests in `tests/gateway/test_<platform>.py`

## Async Notes

- Gateway runs as an **async event loop** — all platform adapters use async/await
- Token locks prevent concurrent startup when using profiles
- `AIAgent` is synchronous — gateway wraps calls with `asyncio.run_in_executor()` or thread dispatch
