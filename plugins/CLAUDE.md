## Module Purpose

Plugin infrastructure for extending Hermes. Currently focused on memory providers, with a general-purpose plugin API for future expansion. Plugins are separate from the skill system -- they provide backend integrations, not procedural knowledge.

## Memory Provider System

- **Only ONE external memory provider active at a time** (configured via `memory.provider` in config.yaml)
- Built-in memory (MEMORY.md / USER.md) is always active and cannot be removed
- External providers are additive, not replacement
- Each provider must contain `__init__.py` implementing the `MemoryProvider` ABC

### MemoryProvider Lifecycle

```
initialize()           -> connect, create resources
system_prompt_block()  -> static text for system prompt
prefetch(query)        -> background recall before each turn
sync_turn(user, asst)  -> async write after each turn
get_tool_schemas()     -> tool schemas to expose to the model
handle_tool_call()     -> dispatch a tool call
shutdown()             -> clean exit
```

### Plugin YAML Metadata

Each plugin can include `plugin.yaml` with description and metadata:

```yaml
description: "Provider description"
```

## Dependencies

- `agent.memory_provider.MemoryProvider` (ABC)
- External SDK per provider (e.g., `honcho-ai` for Honcho)

## Creating a New Memory Provider

1. Create `plugins/memory/<name>/` directory with `__init__.py`
2. Implement `MemoryProvider` ABC from `agent/memory_provider.py`
3. Required methods: `initialize()`, `system_prompt_block()`, `prefetch()`, `sync_turn()`, `get_tool_schemas()`, `handle_tool_call()`, `shutdown()`
4. Optionally add `plugin.yaml` with `description` field
5. Add SDK dependency to `pyproject.toml` under `[project.optional-dependencies]`

## Context Engine

`plugins/context_engine/` is reserved for future context engine plugins. Currently contains only `__init__.py`.
